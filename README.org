* 项目概述

本项目是一个跨平台的自动化运维 / 装机工具，核心功能为"一键装机与配置还原"。用户可用它备份和恢复软件、配置文件以及运行初始化脚本，支持 Windows、macOS、Linux 三个平台。

目前提供终端 TUI（基于 termbox2）作为主交互界面。

** 文件操作注意事项

- ~sqlite/~ 目录下的 ~sqlite3.c~ (9.1 MB) 和 ~sqlite3.h~ (675 KB) 文件巨大，Agent 不应读取其内容，否则会严重浪费上下文。读取 ~sqlite3.h~ 中的某个 API 签名时，应使用 ~grep~ 精确搜索。
- ~shell.c~ (1.1 MB) 是 SQLite CLI shell 源码，本项目不使用，不应读取。
- ~termbox2.h~ (~500 KB) 是单头文件 TUI 渲染库，Agent 可能不需要读取其内容（通过 ~tb_printf~ / ~tb_event_t~ / ~tb_poll_event~ 等 API 使用即可）。如需要查 API 签名，应使用 ~grep~ 。

** 开发语言与风格

- 语言：C11（标准 C，非 C++）
- 命名：函数名使用小驼峰（ ~initDatabase, openURL, renderTuiScene~ ），常量使用 ~k~ 前缀（ ~kCreateTableSQL~ ），宏使用全大写
- 注释：中文注释
- 返回值约定：成功返回 1，失败返回 0
- 禁止使用 ~nullptr~ 、 ~std::string~ 等 C++ 特性

** 代码修改守则

1. 始终遵循"平台无关代码放根目录，平台差异代码放 ~platform/~ " 的原则
2. 数据库 Schema 在 ~init.c~ 的 ~kCreateTableSQL~ 常量字符串中定义，而非单独的文件
3. ~PRAGMA foreign_keys = ON~ 已在 ~init.c~ 中单独执行，不得挪回 SQL 字符串中
4. 不要在 ~sqlite/~ 目录中创建或修改任何文件
5. TUI 的 View（渲染层）禁止 ~#include "sqlite/sqlite3.h"~ ，禁止直接读写 ~sqlite3~ 句柄——它只能读 ~g_tui~ 缓存
6. ~TB_IMPL~ 必须且只能在 ~tui.c~ 中定义一次（其他文件 ~#include "termbox2.h"~ 时不应定义 ~TB_IMPL~）

* TUI 终端界面

** 架构：MVC 模式

TUI 层采用经典 MVC 分层，通过 ~TuiState~ 全局单例 ~g_tui~ 解耦：

- **Model**（~tui.h~）： ~TuiCategory~ , ~TuiApp~ , ~TuiAppConfig~ 三种扁平展示结构体 + ~TuiState~ 状态机
- **View**（~tui.c~ 渲染函数）： ~renderTuiScene()~ 及以下 ~draw*~ 子函数——只读 ~g_tui~ ，纯绘图
- **Controller**（~tui.c~ 事件循环）： ~runTuiLoop()~ 中的 ~tb_poll_event~ 循环 + ~handleKeyEvent()~ ——写 ~g_tui~ 并触重绘

** 布局

程序启动后进入三栏布局的 TUI 浏览模式：

```
+--- 分类列表 ---+--- 软件列表 ------+
| 常用           | [X] Google        |
| 开发工具       | [ ] VS Code       |  ← 焦点绿色高亮
| 聊天通信       | [ ] iTerm2        |
+----------------+-------------------+

|                | Configs: 3        |
+----------------+-------------------+
[macOS] ↑↓ Navigate  Tab Switch  Space/Enter Check  Ctrl+X Deploy  / Search  q/ESC Quit
```

- **左栏**：分类列表（选中行在焦点下显示绿底黑字高亮）
- **右栏上**：当前分类下的软件列表， ~[X]~ / ~[ ]~ 标记勾选状态
- **右栏下**：当前选中软件的详情（名称、平台、配置数量）
- **底部状态栏**：当前平台标签 + 全键盘快捷键提示

** 覆盖模式

- **Deploy（一键装机）**： ~Ctrl+X~ 触发。全屏 ASCII 看板，实时滚动显示每条勾选软件的部署日志（ ~[START]→[DONE]/[FAIL]→[FINISH]~ ），日志前缀按颜色区分：青色=进行中、绿色=成功、红色=失败、黄色=结束。部署过程中拦截所有键盘输入，完成后任意键返回浏览。
- **Search（搜索过滤）**： ~/~ 触发。底部状态栏切换为搜索输入栏，按软件名称实时过滤右侧列表， ~Enter~ 确认， ~ESC~ 退出。
- **Input（输入弹窗）**： ~a~ 触发。居中 ASCII 弹窗，输入分类或软件名后 ~Enter~ 写入数据库并立即刷新前端， ~ESC~ 取消。
- **Confirm（确认弹窗）**： ~d~ 触发。居中 ASCII 弹窗，显示待删除项目名称， ~Y~ / ~Enter~ 确认删除并刷新， ~N~ / ~ESC~ 取消。

* 数据库设计

** 设计理念

1. 平台与配置解耦 —— 同一软件在不同操作系统下的下载链接、安装包路径、配置文件路径和初始化脚本均不相同，因此采用"1 张主表 + 多张子表"结构，避免单张宽表。
2. 规范性与扩展性 —— 分类采用字典表管理，支持用户自定义标签；通过外键约束保证数据一致性。
3. SQLite 特性优化 —— 主键统一使用 ~INTEGER PRIMARY KEY AUTOINCREMENT~ ，布尔值用 ~INTEGER~ 的 0/1 表示。

** 表关系

#+BEGIN_SRC text
categories (分类字典)
  │ (1:N)
  ▼
apps (软件主表)
  │ (1:N)
  ▼
app_platforms (平台与下载)
  │ (1:N)
  ▼
app_configs (配置与脚本)
#+END_SRC

** 建表语句

建表 SQL 全部定义在 ~init.c~ 的 ~kCreateTableSQL~ 常量字符串中，以下为各表结构的参考说明：

*** ~categories~ （分类字典表）

| 字段   | 类型      | 约束                        |
|--------+-----------+-----------------------------|
| ~id~   | ~INTEGER~ | ~PRIMARY KEY AUTOINCREMENT~ |
| ~name~ | ~TEXT~    | ~NOT NULL, UNIQUE~          |

软件分类名称，如"文件管理"、"聊天通信"。

*** ~apps~ （软件主表）

| 字段          | 类型      | 约束                                           |
|---------------+-----------+------------------------------------------------|
| ~id~          | ~INTEGER~ | ~PRIMARY KEY AUTOINCREMENT~                    |
| ~name~        | ~TEXT~    | ~NOT NULL, UNIQUE~                             |
| ~category_id~ | ~INTEGER~ | ~REFERENCES categories(id) ON DELETE SET NULL~ |
| ~priority~    | ~INTEGER~ | ~NOT NULL DEFAULT 0~                           |
| ~description~ | ~TEXT~    |                                                |

~priority~ 控制装机时的安装顺序，数值越小越优先（0 = 默认）。外键 ~ON DELETE SET NULL~ 使得删除分类不影响已有软件记录。

*** ~app_platforms~ （软件平台与下载子表）

| 字段                       | 类型      | 约束                                                                  |
|----------------------------+-----------+-----------------------------------------------------------------------|
| ~id~                       | ~INTEGER~ | ~PRIMARY KEY AUTOINCREMENT~                                           |
| ~app_id~                   | ~INTEGER~ | ~NOT NULL, REFERENCES apps(id) ON DELETE CASCADE~                     |
| ~platform~                 | ~TEXT~    | ~NOT NULL, CHECK (platform IN ('Windows','macOS','Linux','FreeBSD'))~ |
| ~is_manual~                | ~INTEGER~ | ~NOT NULL DEFAULT 0, CHECK (is_manual IN (0, 1))~                     |
| ~download_url~             | ~TEXT~    |                                                                       |
| ~installer_path~           | ~TEXT~    |                                                                       |
| ~UNIQUE(app_id, platform)~ |           | 联合唯一约束                                                           |

~is_manual~ : 0 = 自动下载，1 = 手动安装。 ~UNIQUE~ 防止同一软件在同一个平台重复添加（该系统自动生成联合索引 ~(app_id, platform)~ ，因此无需再为 ~app_id~ 建单列索引）。

*** ~app_configs~ （配置文件与脚本子表）

| 字段              | 类型      | 约束                                                       |
|-------------------+-----------+------------------------------------------------------------|
| ~id~              | ~INTEGER~ | ~PRIMARY KEY AUTOINCREMENT~                                |
| ~app_platform_id~ | ~INTEGER~ | ~NOT NULL, REFERENCES app_platforms(id) ON DELETE CASCADE~ |
| ~config_path~     | ~TEXT~    |                                                            |
| ~script_type~     | ~TEXT~    |                                                            |
| ~script_path~     | ~TEXT~    |                                                            |

记录每个平台版本对应的配置备份路径与初始化脚本（如 Shell、Bat、PowerShell）。

| 索引名                     | 所在表          | 列                   | 说明                                     |
|----------------------------+-----------------+----------------------+------------------------------------------|
| ~idx_apps_category~        | ~apps~          | ~category_id~        | 加速按分类查询软件                         |
| ~idx_app_platforms_plat~   | ~app_platforms~ | ~platform~           | 加速按平台筛选软件                         |

* 关键技术决策

** ~PRAGMA foreign_keys~ 必须在连接后单独执行

#+BEGIN_SRC c
sqlite3_exec(db, "PRAGMA foreign_keys = ON;", NULL, NULL, NULL);
#+END_SRC

~PRAGMA foreign_keys = ON~ 不能与 ~CREATE TABLE~ 揉在同一个 ~sqlite3_exec~ 批处理中，否则可能因 SQLite 内部机制被静默忽略。已在 ~init.c~ 中保证： ~open~ 之后、建表之前单独执行。

** ~is_manual~ 加 ~CHECK~ 约束

~is_manual INTEGER NOT NULL DEFAULT 0 CHECK (is_manual IN (0, 1))~ \\
在列级别限制只能为 0 或 1，防止代码层写入 2、100 等非法值。

** ~UNIQUE~ 自动生成索引

~UNIQUE(app_id, platform)~ 在应用层防止数据重复，同时 SQLite 自动为其创建联合索引。该索引的左前缀 (~app_id~) 已能覆盖按 ~app_id~ 的查询，无需再为 ~app_id~ 建单列索引。

** ~priority~ 字段控制安装顺序

~apps.priority~ 数值越小越优先安装，可用于表达隐式依赖关系（如先装 Node.js 再装依赖它的脚本工具）。

** TUI 使用 ASCII 边框而非 UTF-8 线框字符

为避免终端编码兼容性问题，所有面板边框统一使用 ~+~ 、 ~-~ 、 ~|~ 绘制，而非 ~┌~ ~┐~ ~└~ ~┘~ 等 Unicode 线框字符。

** TUI 纯 View 层禁止接触 SQLite

~renderTuiScene()~ 及其 ~draw*~ 子函数只从 ~g_tui~ 全局缓存中读取数据，不得 ~#include "sqlite/sqlite3.h"~ 或持有 ~sqlite3*~ 句柄。Controller 层负责从 SQLite 查询并填入 ~g_tui~。
